Microsoft 安全公告 MS10-074 - 警告
==================================

Microsoft 基础类中的漏洞可能允许远程执行代码 (2387149)
------------------------------------------------------

发布时间: 2010年10月12日

**版本:** 1.0

### 一般信息

#### 摘要

此安全更新可解决 Microsoft 基础类 (MFC) 库中的一个公开披露的漏洞。 如果用户使用管理用户权限登录并打开使用 MFC 库构建的应用程序，则此漏洞可能允许远程执行代码。 成功利用此漏洞的攻击者可以获得与当前登录用户相同的权限。 如果用户使用管理用户权限登录，攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。

对于 Microsoft Windows 所有受支持的版本，此安全更新的等级为“中等”。 有关详细信息，请参阅本节中“**受影响和不受影响的软件**”小节。

此安全更新通过对窗口标题更改请求增加附加检查来解决漏洞。 有关漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的常见问题 (FAQ) 小节。

**建议。** 大多数客户均启用了“自动更新”，他们不必采取任何操作，因为此安全更新将自动下载并安装。 尚未启用“自动更新”的客户必须检查更新，并手动安装此更新。 有关自动更新中特定配置选项的信息，请参阅 [Microsoft 知识库文章 294871](http://support.microsoft.com/kb/294871)。

对于管理员、企业安装或者想要手动安装此安全更新的最终用户，Microsoft 建议客户考虑使用更新管理软件应用此更新或者使用 [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) 服务检查更新。

另请参阅本公告后面部分中的“**检测和部署工具及指导**”一节。

**已知问题。** 无

Windows MFC 文档标题更新缓冲区溢出漏洞 - CVE-2010-3227  
------------------------------------------------------
  
  
在使用 Microsoft 基础类 (MFC) 库写入的应用程序中管理窗口标题的方式中存在一个远程执行代码漏洞。 尽管此漏洞位于 MFC 中且存在于受影响的操作系统上，但它只能在远程攻击者可以影响 MFC 应用程序中任何窗口的窗口标题时才能被利用。 成功利用此漏洞的攻击者可以在当前用户的安全上下文中运行任意代码。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227)。
  
#### Windows MFC 文档标题更新缓冲区溢出漏洞 (CVE-2010-3227) 的缓解因素
  
Microsoft 并未发现此漏洞的任何缓解因素。
  
#### Windows MFC 文档标题更新缓冲区溢出漏洞 (CVE-2010-3227) 的变通办法
  
Microsoft 并未发现此漏洞的任何变通办法。
  
#### Windows MFC 文档标题更新缓冲区溢出漏洞 (CVE-2010-3227) 的常见问题
  
**此漏洞的影响范围有多大？**  
此漏洞是当前登录用户的安全上下文中的远程执行代码漏洞。
  
**造成此漏洞的原因是什么？**  
此漏洞是由 MFC 处理重命名窗口标题的应用程序请求的方式所导致的。
  
**什么是 Microsoft 基础类 (MFC) 库？**  
Microsoft 基础类 (MFC) 库是用于在 Microsoft Windows 中编程的应用程序框架。 用 C++ 编写的 MFC 提供执行标准操作（例如管理窗口、菜单和对话框），执行基本输入/输出，或存储数据对象集合所需的大部分代码。
  
**即使我的系统上没有合适的攻击媒介，也会为我提供此更新吗？**  
是的，此更新将提供给运行“受影响的软件”部分中概述的软件的所有系统。 如果系统上当前未安装 MFC 应用程序，Microsoft 建议用户安装此更新作为纵深防御措施，以防引入攻击媒介或在以后成为已知对象。
  
**是否有可用作攻击媒介的任何 Microsoft 应用程序？**  
Microsoft 已仔细调查所有应用程序，但未发现可成功用作攻击媒介的 Microsoft 应用程序。
  
**Microsoft 是否获悉可用作攻击媒介的任何第三方 MFC 应用程序？**  
是。 Microsoft 获悉一个可用作攻击媒介的第三方应用程序。 使用此第三方攻击媒介成功利用此漏洞将需要用户使用 MFC 应用程序打开特制文件，然后双击 MFC 应用程序内的文件夹。
  
**我已使用 Microsoft 基础类编写了一个应用程序。 我的应用程序可以用作攻击媒介吗？**  
如果满足以下所有条件，则您的应用程序可能是攻击媒介：
  
-   您的应用程序利用 Microsoft 基础类 (MFC) 库  
-   您的应用程序根据可由远程用户控制的信息设置窗口标题
  
**注意** Microsoft 建议将 MFC 应用程序动态链接到随 Windows 附带的 MFC 版本。 如果您的应用程序附带 MFC 的易受攻击版本并以静态方式与其链接，强烈建议您更新您的应用程序以动态链接到随 Windows 附带的 MFC 版本。
  
**攻击者可能利用此漏洞执行什么操作？**  
成功利用此漏洞的攻击者可以获得与当前登录用户相同的权限。 如果用户使用管理用户权限登录，攻击者便可完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。 那些帐户被配置为拥有较少系统用户权限的用户比具有管理用户权限的用户受到的影响要小。
  
**攻击者如何利用此漏洞？**  
为成功利用此漏洞，目标系统将需要安装可用于攻击媒介的 MFC 应用程序。 可用作攻击媒介的 MFC 应用程序将需要为远程用户提供一种影响 MFC 应用程序内窗口标题的方式。 例如，根据使用 MFC 应用程序打开的文件内容进行的窗口标题更改，或者根据网络输入（如，在聊天应用程序或 Web 浏览器中）进行的窗口标题更改。 根据特定攻击媒介，攻击者可能需要诱使用户使用用户系统上安装的特定 MFC 应用程序打开特制文件，并执行特定操作。 在其他攻击媒介中，如聊天应用程序，攻击者只需更改聊天室名称可能就足够了，或者与用户联系时采用特制处理。
  
**受此漏洞威胁最大的系统有哪些？**  
如果安装 MFC 应用程序允许远程用户影响 MFC 应用程序内的窗口标题，则工作站和终端服务器受到的威胁最大。 如果管理员允许用户登录服务器并运行此类 MFC 程序，服务器可能面临更大风险。 不过，最佳做法是不要授予这样的权限。
  
**此更新有什么作用？**  
此更新通过对窗口标题更改请求增加附加检查来解决漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**  
是。 此漏洞已公开披露。 已为此漏洞分配了常见漏洞和披露号码 [CVE-2010-3227](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3227)。
  
**在发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**  
否。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已公开用于攻击用户的信息。

管理需要部署到组织中的服务器、台式机和移动系统的软件和安全更新。 有关详细信息，请参阅 [TechNet 更新管理中心](http://go.microsoft.com/fwlink/?linkid=69903)。 [Microsoft TechNet 安全网站](http://go.microsoft.com/fwlink/?linkid=21132)提供了有关 Microsoft 产品安全的详细信息。

最后，可以从 [Microsoft Update 目录](http://go.microsoft.com/fwlink/?linkid=96155)下载安全更新。 Microsoft Update 目录提供通过 Windows Update 和 Microsoft Update 提供的内容的可搜索目录，包括安全更新、驱动程序和 Service Pack。 通过使用安全公告编号（例如“MS07-036”）进行搜索，您可以将所有适用的更新添加到您的篮（包括某个更新的不同语言），然后将其下载到您选择的文件夹。 有关 Microsoft Update 目录的详细信息，请参阅 [Microsoft Update 目录常见问题](http://go.microsoft.com/fwlink/?linkid=97900)。

#### 部署信息

**安装更新**

当您安装此安全更新时，安装程序将检查系统中正在更新的一个或多个文件是否已由 Microsoft 修补程序更新。

如果以前安装的修补程序已经更新过其中的某个文件，安装程序就会将 RTMQFE、SP1QFE 或 SP2QFE 文件复制到系统中。 否则，安装程序将 RTMGDR、SP1GDR 或 SP2GDR 文件复制到您的系统中。 安全更新可能不包含这些文件的所有版本。 有关此行为的详细信息，请参阅 [Microsoft 知识库文章 824994](http://support.microsoft.com/kb/824994)。

有关安装程序的详细信息，请访问 [Microsoft TechNet 网站](http://go.microsoft.com/fwlink/?linkid=38951)。

有关本公告中出现的术语（如“*修补程序*”）的详细信息，请参阅 [Microsoft 知识库文章 824684](http://support.microsoft.com/kb/824684)。

-   **文件版本验证**

#### Microsoft Active Protections Program (MAPP)
  
为改进客户的安全保护，Microsoft 在发布每月安全更新之前将向主要的安全软件供应商提供漏洞信息。 然后，安全软件供应商可以使用该漏洞信息通过其安全软件或者设备向客户提供更新的保护，例如防病毒、基于网络的入侵检测系统或者基于主机的入侵防止系统。 要确定是否可从安全软件供应商处得到活动保护，请访问计划合作伙伴（在 [Microsoft Active Protections Program (MAPP) 合作伙伴](http://www.microsoft.com/security/msrc/mapp/partners.mspx)中列出）提供的活动保护网站。
