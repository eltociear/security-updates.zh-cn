Security Bulletin

Microsoft 安全公告 MS08-037 - 重要
==================================

DNS 中的漏洞可允许欺骗 (953230)
-------------------------------

发布时间: 2008年7月8日 | 更新时间: 2009年12月9日

**版本:** 3.1

###一般信息

####摘要

此安全更新可解决 Windows 域名系统 (DNS) 中两个秘密报告的漏洞，这些漏洞可允许欺骗。 DNS 客户端和 DNS 服务器中均存在这些漏洞，这些漏洞可能会允许远程攻击者将 Internet 上针对系统的网络通信重定向至攻击者自己的系统。

对于 Microsoft Windows 2000、Windows XP、Windows Server 2003、Windows Server 2008 的所有受支持版本，此安全更新等级为“重要”。有关详细信息，请参阅本节的**受影响和不受影响的软件**小节。

此安全更新可消除该漏洞，方法是使用随机程度较高的 DNS 事务 ID，使用随机套接字进行 UDP 查询，并更新用于管理 DNS 缓存的逻辑。 有关漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的常见问题 (FAQ) 小节。

**建议。** Microsoft 建议客户尽早应用此更新。

**已知问题。** [Microsoft 知识库文章 953230](http://support.microsoft.com/kb/953230) 介绍了客户在安装此安全更新时可能遇到的当前已知问题。

####受影响和不受影响的软件

已对下列软件进行测试，以确定受到影响的版本。 其他版本的支持生命周期已结束或者不受影响。 要确定软件版本的技术支持生命周期，请访问 [Microsoft 技术支持生命周期](http://go.microsoft.com/fwlink/?linkid=21742)。

与此安全更新相关的常见问题 (FAQ)  
--------------------------------
    
**为什么在 2009 年 12 月 8 日重新发布此公告？**      
Microsoft 重新发布此公告，以便重新提供 Microsoft Windows 2000 Service Pack 4 上的 DNS 客户端的更新 (KB951748)，为其他代码路径提供随机程度较高的 DNS 事务 ID。 与其他 Windows 平台不同的是，Microsoft Windows 2000 上存在两条 DNS 事务代码路径。 以前的更新仅在代码路径之一上提供事务 ID 随机化。 此更新的重新发布版本为 Microsoft Windows 2000 上的其他代码路径提供相同的事务 ID 随机化。之前已经在 Microsoft Windows 2000 Service Pack 4 上安装了 DNS 客户端更新 (KB951748) 的客户需要安装自动重新提供的更新。 没有其他更新受此重新发布版本的影响。
  
**为什么在 2009 年 1 月 13 日修订此安全公告？**  
Microsoft 修订了此公告，阐明此公告提供的更新可能未正确地提供给运行 Windows XP Service Pack 3 的所有系统。检测和部署问题已经解决，并即将正确地向尚未应用此公告提供的更新的使用 Windows XP Service Pack 3 系统的客户提供此更新。 没有更改此更新的二进制文件。 成功地更新了其系统的客户不需要重新安装此更新。
  
**为什么在 2008 年 7 月 25 日修订此安全公告？**  
Microsoft 修订了此安全公告，向“**与此安全更新相关的常见问题 (FAQ)**”添加了三个新的已知问题。 这些常见问题如下：
  
-   **客户在安装此更新之后可能遇到的端口冲突问题是什么？**  
    如果 DNS 服务器执行其他服务，这些服务需要静态 UDP 端口，这些端口在 DNS 服务器为其套接字池分配的端口范围内，那么 DNS 服务器可能会遇到端口冲突而导致服务失败。 有关详细信息，[Microsoft 知识库文章 953230](http://support.microsoft.com/kb/953230) 说明了客户在安装此安全更新时可能遇到的当前已知问题。  
-   **SBS 客户在安装此更新之后可能遇到的问题是什么？**  
    Windows Small Business Server (SBS) 2003 的受支持版本包含与 Windows Server 2003 Service Pack 1 和 Windows Server 2003 Service Pack 2 相同的受影响代码。但是，SBS 系统在安装此公告中提供的 DNS 服务器安全更新之后可能遇到服务失败或无法正常工作。 有关详细信息，[Microsoft 知识库文章 953230](http://support.microsoft.com/kb/953230) 说明了客户在安装此安全更新时可能遇到的当前已知问题。  
-   **外围防火墙和外围 NAT 设备在安装此更新之后可能遇到的问题是什么？**  
    在基于 Windows 的计算机上安装此公告提供的安全更新之后，来自跨越防火墙的主机的 DNS 查询可能无法再使用随机源端口。 有关详细信息，[Microsoft 知识库文章 953230](http://support.microsoft.com/kb/953230) 说明了客户在安装此安全更新时可能遇到的当前已知问题。
  
**为什么在 2008 年 7 月 23 日修订此安全公告？**  
Microsoft 已修订此安全公告，将 MS06-064、MS07-062 和 MS08-001 列为此更新替代的公告。
  
**为什么在 2008 年 7 月 10 日修订此安全公告？**  
Microsoft 修订此安全公告旨在通知使用 [Check Point Software Technologies Ltd.](http://www.checkpoint.com/) 的 ZoneAlarm 和 Check Point Endpoint Security（以前称为 Check Point Integrity）的用户，有关以下常见问题中详细说明的 Internet 连接问题。 修订版本不会更改本公告中的安全更新文件。 已经应用了更新的客户不必重新应用该更新。 但是，使用 Check Point 提供的上述软件的用户应阅读以下常见问题以获得进一步指导。
  
**安装此更新后，ZoneAlarm 和 Check Point Endpoint Security 的用户可能会遇到什么问题？**  
Microsoft 获悉的最新报告显示，使用 Check Point Software Technologies Ltd. 提供的 ZoneAlarm 和 Check Point Endpoint Security（以前称为 Check Point Integrity）的用户在应用本安全公告 MS08-037 提供的安全更新后遇到 Internet 连接问题。Microsoft 将与 Check Point 继续调查此问题。 Microsoft 鼓励 [ZoneAlarm](http://download.zonealarm.com/bin/free/pressreleases/2008/lossofinternetaccessissue.html) 和 Check Point Endpoint Security 的用户查看适当的 [Check Point](https://supportcenter.checkpoint.com/supportcenter/index.jsp) 网站和本安全公告，以获得最新指导或软件更新。

**安装此安全更新后，客户可能会遇到哪些卸载问题？**  
受支持版本的 Microsoft Windows 2000 和 Windows 2003 系统将收到 DNS 服务器和 DNS 客户端更新。 DNS 服务器更新和 DNS 客户端更新共享二进制文件，卸载顺序必须与安装顺序相反，以避免共享的二进制文件回归到之前的版本。 [Microsoft 知识库文章 823836](http://support.microsoft.com/kb/823836) 介绍了客户在卸载这些安全更新时可能遇到的当前已知问题。
  
**客户在安装此安全更新时可能遇到哪些随机套接字连接问题？**  
默认情况下，此安全公告提供的 DNS 更新将利用大量的可用套接字并提供最多平均信息量。 但是，如果用户在注册表中定义了端口范围，更新将使用用户定义的设置，仅分配定义的套接字。
  
可以在下列注册表位置定义套接字范围：  
**HKLM\\System\\CurrentControlSet\\Services\\DNS\\Parameters Reg key Name: SocketPoolSize**  
**注意** 必须重新启动 DNS 服务才能实施这些更改。
  
**定义套接字池范围是什么？**  
定义 DNS 能够从其中选择的套接字范围可能是避免与其他在通信时需要相同套接字池的应用程序或服务发生冲突所必需的。 请参阅 [MaxUserPort](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/58791.mspx?mfr=true) 和 [Microsoft 知识库文章 812873](http://support.microsoft.com/kb/812873) 以了解有关这些注册表项注册的更多信息。
  
**定义套接字池范围时操作系统之间的差异是什么？**  
MaxUserPort 注册表项在 Windows Vista 和 Windows Server 2008 上的含义与其在 Microsoft Windows Server 2000 和 Windows Server 2003 上的含义不同。[Microsoft 知识库文章 929851](http://support.microsoft.com/kb/929851) 详细说明了 Windows Vista 和 Windows Server 2008 在操作上的变化。
  
在 Microsoft Windows Server 2000 和 Windows Server 2003 中，设置 MaxUserPort 定义了动态端口范围的结束点。 范围始于 1024，一直继续到用户在 MaxUserPort 注册表项设置中定义的值。 在安装由安全公告提供的更新后，Microsoft Windows Server 2000 和 Windows Server 2003 上的默认行为将从 49152 至 65535 的端口范围随机分配套接字。如果已定义 MaxUserPort 范围，则将从 1024 到在 MaxUserPort 注册表项设置中定义的值之间随机分配端口。 有关在 Microsoft Windows 2000 服务器和 Windows Server 2003 上保留端口范围的详细信息，请访问 [Microsoft 知识库文章 812873](http://support.microsoft.com/kb/812873)。
  
在 Windows Vista 和 Windows Server 中，设置 MaxUserPort 定义了动态端口范围的起点。 默认情况下，Windows Vista 和 Windows Server 2008 上的范围是 49152 至 65535。

DNS 套接字熵不足的漏洞 - CVE-2008-1447  
--------------------------------------
    
在 Windows DNS 客户端和 Windows DNS 服务器内存在一个欺骗漏洞。 此漏洞可允许未经身份验证的远程攻击者迅速可靠地欺骗响应并将记录插入 DNS 服务器或者客户端缓存，从而重定向 Internet 通信量。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2008-1447](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1447)。
  
####DNS 套接字熵不足的漏洞 (CVE-2008-1447) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。
  
-   在 TCP 和 IP 层上运行的加密协议，例如 IPsec 或者 SSL/TLS，可以阻止攻击者用重定向的通信量进行监控或者干扰。
  
####DNS 套接字熵不足的漏洞 (CVE-2008-1447) 的变通方法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。
  
-   Microsoft 并未发现此漏洞的任何变通办法。
  
####DNS 套接字熵不足的漏洞 (CVE-2008-1447) 的常见问题
  
**此漏洞的影响范围有多大？**      
在 Windows DNS 客户端和 DNS 服务器内存在一个欺骗漏洞。 成功利用此漏洞的攻击者能够将任意地址插入 DNS 缓存。
  
**造成漏洞的原因是什么？**      
Windows DNS 客户端和 DNS 服务器中的 Windows DNS 服务在执行 DNS 查询时没有提供足够的熵。
  
**什么是域名系统 (DNS)？**  
域名系统 (DNS) 是一套包含 TCP/IP 的行业标准协议的其中之一。 DNS 是通过使用两个软件组件来实现的： DNS 服务器和 DNS 客户端（或者解析器）。 两个组件是作为背景服务应用程序运行。 网络资源是由数字 IP 地址标识，但是这些 IP 地址让网络用户很难记住。 DNS 数据库包含将网络资源的用户友好字母数字名称（例如 www.microsoft.com）映射到这些资源用于通信的 IP 地址的记录。 因此，DNS 便充当一个助记键设备，使网络资源更容易被网络用户记住。 有关更多信息以及查看说明 DNS 如何适合其他 Windows 技术的逻辑图表，请阅读文章“[什么是 DNS](http://technet2.microsoft.com/windowsserver/en/library/ff937311-03ce-4d04-b72c-b39c4d51cb361033.mspx)”。
  
**DNS 缓存是什么？**  
域名系统 (DNS) 缓存解析程序服务是保存 DNS 查询响应的一项服务，这样 DNS 服务器就不必重复查询相同的信息。 有关详细信息，请参阅 [DNSCache](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/regentry/30643.mspx?mfr=true) Technet 文章或者 [DNS 解析程序缓存服务](http://www.microsoft.com/technet/prodtechnol/windows2000serv/reskit/cnet/cnbc_imp_qxht.mspx?mfr=true) Technet 文章。 有关 DNS 缓存中毒的更多信息，请参阅[攻击检测](http://www.microsoft.com/technet/isa/2004/help/fw_alertattack.mspx?mfr=true)的 TechNet 文章。
  
**攻击者可能利用此漏洞执行什么操作？**      
成功利用此漏洞的攻击者能够将任意地址插入 DNS 缓存，又称 DNS 缓存中毒。
  
**攻击者如何利用此漏洞？ **  
攻击者可以将特定查询发往脆弱的 DNS 服务器或客户端，同时以允许攻击者插入错误或误导性的 DNS 数据的方式发回响应。 攻击者随后可以将 Internet 通信量从合法位置重定向到攻击者选择的地址。
  
**受此漏洞威胁最大的系统有哪些？**      
连接到网络或 Internet 的任何 Windows 系统都将受到威胁。 Windows Vista 和 Windows Server 2008 不受此漏洞的影响。
  
**此更新有什么作用？ **  
此更新通过使用随机程度较高的 DNS 事务 ID 值和随机的 UDP 套接字进行远程查询来解决这个漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**      
否。 Microsoft 通过可靠的披露渠道了解到有关此漏洞的信息。
  
**发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**      
否。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已被公开用于攻击客户的信息，也没有看到任何发布的概念代码证明示例。
  
DNS 缓存中毒漏洞 - CVE-2008-1454  
--------------------------------
    
Windows DNS 服务器中存在一个缓存中毒漏洞。 此漏洞可允许未经身份验证的远程攻击者将特制的响应发送给易受攻击的系统的 DNS 请求，从而使 DNS 缓存中毒，并将 Internet 通信量从合法位置重定向至其他位置。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2008-1454](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1454)。

**验证是否已应用更新**

-   **Microsoft Baseline Security Analyzer**

    要验证受影响的系统上是否应用了安全更新，可以使用 Microsoft Baseline Security Analyzer (MBSA) 工具。 有关详细信息，请参阅本公告前面的“**检测和部署工具及指导**”一节。

-   **文件版本验证**

    由于 Microsoft Windows 有多个版本，因此以下步骤可能会与您的系统上的步骤不同。 如果不同，请参阅产品文档来完成这些步骤。

    1.  单击“**开始**”，然后单击“**搜索**”。
    2.  在“**搜索结果**”窗格中，单击“**搜索助理**”下的“**所有文件和文件夹**”。
    3.  在“**完整或部分文件名**”框中，键入相应文件信息表中的一个文件名，然后单击“**搜索**”。
    4.  在文件列表中，用鼠标右键单击相应文件信息表中的一个文件名，然后单击“**属性**”。
        **注意** 根据系统上安装的操作系统或程序的版本，文件信息表中所列的某些文件可能并未安装。
    5.  在“**版本**”选项卡上，通过将系统上安装的文件与相应文件信息表中记录的版本进行比较来确定该文件的版本。
        **注意** 在安装过程中，除文件版本以外的其他属性均可能会发生变化。 不支持通过将其他文件属性与文件信息表中的信息进行比较这种方法来验证是否应用了更新。 另外，在某些情况下，在安装过程中还可能会重命名文件。 如果没有文件或版本信息，请使用另外一种方法来验证更新安装。

-   **注册表项验证**

    您也可以通过查看本节“**参考表**”中列出的注册表项来验证此安全更新安装的文件。

    这些注册表项可能不包含已安装文件的完整列表。 另外，如果管理员或 OEM 将此安全更新集成或组合到 Windows 安装源文件中，则可能无法正确创建这些注册表项。

**验证是否已应用更新**

-   **Microsoft Baseline Security Analyzer**

    要验证受影响的系统上是否应用了安全更新，可以使用 Microsoft Baseline Security Analyzer (MBSA) 工具。 有关详细信息，请参阅本公告前面的“**检测和部署工具及指导**”一节。

-   **文件版本验证**

    由于 Microsoft Windows 有多个版本，因此以下步骤可能会与您的系统上的步骤不同。 如果不同，请参阅产品文档来完成这些步骤。

    1.  单击“**开始**”，然后单击“**搜索**”。
    2.  在“**搜索结果**”窗格中，单击“**搜索助理**”下的“**所有文件和文件夹**”。
    3.  在“**完整或部分文件名**”框中，键入相应文件信息表中的一个文件名，然后单击“**搜索**”。
    4.  在文件列表中，用鼠标右键单击相应文件信息表中的一个文件名，然后单击“**属性**”。
        **注意** 根据系统上安装的操作系统或程序的版本，文件信息表中所列的某些文件可能并未安装。
    5.  在“**版本**”选项卡上，通过将系统上安装的文件与相应文件信息表中记录的版本进行比较来确定该文件的版本。
        **注意** 在安装过程中，除文件版本以外的其他属性均可能会发生变化。 不支持通过将其他文件属性与文件信息表中的信息进行比较这种方法来验证是否应用了更新。 另外，在某些情况下，在安装过程中还可能会重命名文件。 如果没有文件或版本信息，请使用另外一种方法来验证更新安装。

    您也可以通过查看本节“**参考表**”中列出的注册表项来验证此安全更新安装的文件。

    这些注册表项可能不包含已安装文件的完整列表。 另外，如果管理员或 OEM 将此安全更新集成或组合到 Windows 安装源文件中，则可能无法正确创建这些注册表项。

|-------------------|------------------------------------------------------------------------------------------------|

| **/quiet**        | 抑制显示状态或者错误消息。                                                                     |  
| **/norestart**    | 当与 **/quiet** 结合使用时，系统在安装之后不会重新启动，即使要求重新启动才能完成安装也是如此。 |
