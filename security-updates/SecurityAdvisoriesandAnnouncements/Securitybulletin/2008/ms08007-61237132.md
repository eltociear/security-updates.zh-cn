Security Bulletin

Microsoft 安全公告 MS08-007 - 严重
==================================

WebDAV Mini-Redirector 中的漏洞可能允许远程执行代码 (946026)
------------------------------------------------------------

发布时间: 2008年2月12日 | 更新时间: 2008年2月13日

**版本:** 1.1

### 一般信息

#### 摘要

此关键安全更新解决了 WebDAV Mini-Redirector 中一个秘密报告的漏洞。 成功利用此漏洞的攻击者可以完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。

这是针对 Windows XP 和 Windows Vista 所有受支持版本的严重安全更新以及 Windows Server 2003 所有受支持版本的重要安全更新。有关详细信息，请参阅本节中的小节“**受影响和不受影响的软件**”。

此安全更新通过修改 Mini-Redirector 处理长路径名称的方式解决该漏洞。 有关漏洞的详细信息，请参阅下一节“**漏洞信息**”下面特定漏洞条目的常见问题 (FAQ) 小节。

**建议。** Microsoft 建议用户立即应用此更新。

**已知问题。** 无

Mini-Redirector 堆溢出漏洞 - CVE-2008-0080  
------------------------------------------
  
<span></span>  
WebDAV Mini-Redirector 处理响应的方式中存在一个远程执行代码漏洞。 成功利用此漏洞的攻击者可以完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建新帐户。
  
要在“常见漏洞和披露”列表中以标准条目查看此漏洞，请参阅 [CVE-2008-0080](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0080)。
  
#### Mini-Redirector 堆溢出漏洞 (CVE-2008-0080) 的缓解因素
  
缓解是指一种设置、通用配置或者一般的最佳实践，它以默认状态存在，能够降低利用漏洞的严重性。 以下缓解因素在您遇到的情形中可能会有所帮助：
  
-   在 Windows Server 2003 中，默认情况下禁用 Web 客户端服务。
  
#### Mini-Redirector 堆溢出漏洞 (CVE-2008-0080) 的变通办法
  
变通办法是指一种设置或配置更改，它不能从根本上纠正漏洞，但有助于在应用更新之前封堵已知的攻击源。 Microsoft 已测试了以下变通方法，并在讨论中指明了变通方法是否会降低功能性：
  
-   **禁用 Web 客户端服务**
  
    要禁用 Web 客户端服务，请按照以下步骤操作：
  
    1.  单击“**开始**”，单击“**运行**”，键入“**Services.msc**”，然后单击“**确定**”。  
    2.  右键单击“**WebClient 服务**”，然后选择“**属性**”。  
    3.  将启动类型更改为“**已禁用**”。 如果服务正在运行，请单击“**停止**”。  
    4.  单击“**确定**”，退出管理应用程序。
  
-   **变通办法的影响。** WebDAV 共享将不可访问。
  
    **如何撤消变通方法。**要撤消变通方法，请执行下列步骤：
  
    1.  单击“**开始**”，单击“**运行**”，键入“**Services.msc**”，然后单击“**确定**”。  
    2.  右键单击“**WebClient 服务**”，然后选择“**属性**”。  
    3.  将启动类型更改为“**自动**”并单击**“应用**”。  
    4.  单击“启动”，然后单击“确定”，退出管理应用程序。
  
#### Mini-Redirector 堆溢出漏洞 (CVE-2008-0080) 的常见问题
  
**此漏洞的影响范围有多大？**    
这是一个远程执行代码漏洞。 成功利用此漏洞的攻击者可以完全控制受影响的系统。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户
  
**造成漏洞的原因是什么？**    
Mini-Redirector 不正确地处理特制的 WebDAV 响应。
  
**什么是 WebDAV？**    
Web 分布式创作和版本管理 (WebDAV) 是超文本传输协议 (HTTP) 的扩展，它定义计算机如何使用 HTTP 执行复制、移动、删除和创建等基本文件功能。
  
**什么是 WebDAV Redirector？**    
WebDAV Redirector 是通过 WebDAV 协议执行的远程文件系统，允许 Windows 客户端计算机通过命令行连接到您的 WebDAV 发布目录。 WebDAV Redirector 使您能够在操作 Web 上的文件时，就好像文件在映射网络驱动器上一样。
  
**什么是 WebDAV Mini-Redirector？**    
WebDAV Mini-Redirector 也称为 Web 客户端服务。 此服务使启用 DAV 的文件夹显示为 通用命名约定 (UNC) 共享。
  
**攻击者可能利用此漏洞执行什么操作？**    
成功利用此漏洞的攻击者可以运行任意代码。 攻击者可随后安装程序；查看、更改或删除数据；或者创建拥有完全用户权限的新帐户。
  
**攻击者如何利用此漏洞？**    
要利用此漏洞，需要进行用户交互。 攻击者可能通过在提供 WebDAV 资源的服务器上配置特制目录，然后说服受影响系统的用户使用命令行通过 Web 客户端服务访问该 WebDAV 资源或者通过网站中的链接访问该 WebDAV 资源来利用该漏洞。 但是在所有情况下，攻击者无法强制用户访问特制的 WebDAV 资源。
  
**受此漏洞威胁最大的系统有哪些？**    
启用了 Web 客户端服务的系统受到的威胁最大。 如果管理员允许用户登录服务器并运行程序，服务器可能面临更大风险。 不过，最佳做法是不要授予这样的权限。
  
**此更新有什么作用？**    
此更新通过正确地验证 WebDAV 响应消除该漏洞。
  
**发布此安全公告时，此漏洞是否已公开披露？**    
否。 Microsoft 通过可靠的披露渠道了解到有关此漏洞的信息。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已公开披露的信息。 此安全公告解决了秘密披露的漏洞以及通过内部调查发现的其他问题。
  
**发布此安全公告时，Microsoft 是否收到任何有关此漏洞已被利用的报告？**    
否。 在最初发布此安全公告时，Microsoft 未收到任何表明此漏洞已被公开用于攻击客户的信息，也没有看到任何发布的概念代码证明示例。

    由于 Microsoft Windows 有多个版本，因此以下步骤可能会与您的系统上的步骤不同。 如果不同，请参阅产品文档来完成这些步骤。

        **注意** 在安装过程中，除文件版本以外的其他属性均可能会发生变化。 不支持通过将其他文件属性与文件信息表中的信息进行比较这种方法来验证是否应用了更新。 另外，在某些情况下，在安装过程中还可能会重命名文件。 如果没有文件或版本信息，请使用另外一种方法来验证更新安装。
